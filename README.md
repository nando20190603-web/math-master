# 数学マスター

中学1年生（4〜7月）を対象とした、ブラウザで動作する数学学習アプリです。
毎日異なる問題が出題され、苦手単元を自動で強化する仕組みを備えています。

---

## 概要

| 項目 | 内容 |
|---|---|
| 対象 | 中学1年生（正の数・負の数 / 文字と式 / 方程式） |
| 問題数 | 全75問（基礎・応用） |
| 1回の学習 | 基礎10問 + 応用10問（計20問） |
| 動作環境 | ブラウザのみ（サーバー不要） |
| データ保存 | ブラウザのlocalStorage（端末ローカル） |

## 機能

- **出題** — 苦手単元の出題数を自動調整するウェイト方式
- **採点** — 数値・文字式・記述など複数の採点モードに対応
- **フィードバック** — 正解・不正解ごとにコメントを表示
- **分析** — 日別正解数グラフ・単元別正解率・次回出題バランスを表示
- **学習履歴** — セッション数・累計正解数をポータルに表示
- **レスポンシブ対応** — PC・タブレット・スマートフォンで動作

## ファイル構成

```
math-master/
└── math-master.html   # HTML・CSS・JavaScript・問題データをすべて含む単一ファイル
```

## 起動方法

`math-master.html` をブラウザで開くだけで動作します。
ローカルサーバーやインストールは不要です。

## 今後の改善候補

チームで方針を検討するための項目です。優先度の目安を示しています。

### 優先度：高

#### 1. 問題のハードコーディング解消

**現状：** 75問すべてがJavaScriptコード内に直接記述されています。
問題の追加・修正にはコードの編集が必要で、エンジニア以外が手を加えにくい状態です。

**改善案：**
- 問題データをJSONまたはCSVファイルに分離する
- 管理用UIを設け、ブラウザから問題を追加・編集できるようにする
- 将来的にはGoogleスプレッドシートやCMSと連携する

---

#### 2. データ消失リスクへの対策

**現状：** 学習データはブラウザのlocalStorageにのみ保存されています。
ブラウザのキャッシュをクリアすると**すべてのデータが消えます**。
また、端末をまたいだデータの共有もできません。

**改善案：**
- データのエクスポート／インポート機能を追加する（最小コストの対策）
- サーバーやクラウドへの保存に切り替える

---

### 優先度：中

#### 3. 採点ロジックの改善

**現状：** 文字式の回答は文字列の一致で判定しているため、数学的に同値でも不正解になるケースがあります。

例：
- 正解 `5a円` に対して `5a` と入力 → 不正解扱い
- 正解 `-6x+15` に対して `15-6x` と入力 → 不正解扱い（数学的には同値）

**改善案：**
- 単位（円、cm² など）の有無を許容する
- 項の順序が違っても正解とする柔軟な判定を実装する

---

#### 4. 複数ユーザーへの対応

**現状：** ユーザーの概念がないため、1台の端末を複数の生徒が使うとデータが混在します。

**改善案：**
- 起動時に名前やIDを選択する画面を追加する
- ユーザーごとにlocalStorageのキーを分けて管理する

---

#### 5. セッション構成のカスタマイズ

**現状：** 1回の学習が「基礎10問＋応用10問」に固定されています。

**改善案：**
- 問題数や基礎／応用の割合を設定できるようにする
- 特定の単元だけを集中的に練習するモードを追加する

---

### 優先度：低

#### 6. ファイル構成の整理

**現状：** HTML・CSS・JavaScript・問題データがすべて1ファイル（約1000行）に集約されています。
複数人での開発や、変更履歴の管理が難しい状態です。

**改善案：**
- `style.css`、`app.js`、`questions.json` などにファイルを分割する
- フレームワーク（Vue / React など）の導入を検討する

---

#### 7. 復習アルゴリズムの強化

**現状：** 苦手単元の出題頻度を上げる仕組みはありますが、「問題単位」での管理はしていません。
忘却曲線を考慮した効果的な復習タイミングの制御ができていません。

**改善案：**
- SM-2などのスペースドリピティションアルゴリズムを問題単位で適用する
